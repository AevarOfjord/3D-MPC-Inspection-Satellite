# Missions

This folder is split into two tiers:

- `missions/examples/`: curated sample missions meant to stay stable for demos.
- `missions/dev/`: user-saved and scratch missions created during development.

The Mission Control UI saves mission JSON files into `missions/dev/`. Unified
missions (v2) are stored separately in `missions_unified/` (for example
`missions_unified/latest_preview.json`).
